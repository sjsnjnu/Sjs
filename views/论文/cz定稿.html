    # 摘要：

    从人机物系统的三个角度挖掘可优化属性，分别从人类社会中的用户情感倾向、工作者行为风格，物理世界中的地理位置和信息系统中的历史记录出发，综合各方面属性建立多目标优化模型；依靠边缘计算环境由近及远的分配策略；采用模型验证的方法验证分析任务分配算法和分配结果的合理性。

    1. 建立群智系统的优化模型
    2. 借助边缘计算环境设计基于边缘计算环境的分配策略
    3. 采用形式化方法对任务分配模型和分配结果进行验证分析

    # 第一章 绪论

    移动设备的便携和计算力使移动群智感知(Mobile Crowd-Sensing简称MCS)系统成为了一种通用平台，作为一种新的服务范式，它利用移动用户的智能设备，来执行任务请求者要求的传感任务。

    任务分配是MCS相关研究的重点之一，目前问题：

    1. 任务分配时，忽略了人为因素，包括工作者和用户
    2. 工作者的选择基于单目标优化问题
    3. 任务分配由云计算中心统一调配

    而人机物融合系统(Human-Cyber-Physical System，简称HCPS)的发展提供了新思路。

    Server Allocation Tasks 服务器分配任务模式

    # 研究现状：

    对于多任务系统中的任务分配问题，主要集中在各种多目标函数和约束条件方面，且多集中于解决单目标问题，主要目标是优化功能属性。

    也有一些研究开始关注任务分配中存在的非功能属性问题，如安全和隐私问题。

    🔏Wang等人提出了一个带有地理混淆的位置隐私保护任务分配框架， 在任务分配期间保护用户的位置[20]Wang L, Yang D, Han X, et al. Location Privacy-Preserving Task Allocation for

    Mobile Crowdsensing with Differential Geo-Obfuscation

    在[19]中，提出了一种新的数据融合隐私保护算法，使得服务器不能得到单个参 与者的数据。

    形式化验证方法：数字信息和物质世界是相互协作并存的。模型检测作为一组用于反应系统自动分析的技术，可以帮助我们适应新模型的需要。偶图是一种通用的形式化建模形式，用于表示实体的空间关系及其全局交互。

    # 第二章 基础理论与🌧️相关技术

    1. 基于深度学习的方法又包括基于单一神经网络的、基于混合神经网络的、基于引入注意力机制的和预训练模型等阶段。

      BERT是一种基于Transformer架构的深层双向语言模型，它舍弃了卷积和循环神经网络的结构，对于不同的应用场景，只需要根据具体任务加入一个额外的输入层并进行微调。

    2. 群智任务分配方法

      任务分配模型中，存在两个主体，即任务模型和工作者模型。任务属性又分为任务属性和任务约束，任务约束主要包括，成本限制，时空约束、质量需求。工作者主要有能力水平和信誉值等属性。

      目前的群智任务分配算法里，存在以下几类方法：

      • 基于推荐算法的分配，（引入协同过滤，但需要大量历史任务数据来感知工作者的特征）
      • 基于矩阵分解的算法（解决推荐算法中矩阵稀疏的问题，通过概率矩阵分解的方法将高纬的工作者-任务矩阵A分解为工作者潜在特征矩阵W和任务特征矩阵T使得A=W·T，这种方式能够预测A中的缺失值，有效解决稀疏性问题）
      • 其他分配算法，如基于隐私保护的分配，对工作者和用户的敏感信息进行加密。又比如组合优化策略。

    3. K-medoids聚类

      K-medoids算法在K-means算法的基础上进行了改进，选取每个聚类中与其他数据差异最小的真实数据作为聚类中心，即K-medoids聚类的簇中心是实际存在的样本点。优点是可以改善K-means算法对噪声点和异常值的鲁棒性

    4. 偶图

      是一种基于图形的形式化工具，BigraphER是一套开源工具，它包含一个OCaml编程语言库

    5. PRISM概率检测器

      偶图主要用于定性检测，即是否存在某种性质。Prism已经为几种类型的概率模型提供了模型检测功能，主要为离散或连续时间马尔可夫链和马尔科夫决策过程，以及用于表达它们的建模语言。

    # 第三章 群智系统的模型建立

    1. 人为因素：

      本文主要考虑人为因素，包括工作者和任务请求者。

    （1）偏好分析

    本文使用预训练模型，ALBERT具备许多优点，且提供了对中文的支持，本文根据精简了BERT参数数量的ALBERT模型作为预训练模型进行情感分类。文本情感分析可以分为以下几个关键步骤：

    • 分词 因为神经网络训练无法直接处理整段的文字
    • 词向量化： 经过分词的词片段需要经过向量化，才能放入神经网络中训练，词向量化提供了一种数学化的方法，把自然语言这种符号信息转化为向量形式的数字信息
    • 分数映射： 对于每一条评论，分类模型的输出为该文本的情感倾向为正面的概率，经过多次试验🧪后将概率与得分建立出映射表

    （2） 风险预测

    为了选择安全高效的工作者，需要评估工作者的风险等级

    本文以滴滴盖亚开放计划提供的盖亚数据集作为工作者的驾驶行为为依据

    1. 建立模型

      实验分析：

      1. 数据集： 以收集的评论数据为基础融合后进行试验。

      2. 对比方法：为了验证这种多目标优化分配算法的效果，设计了两种基准方法AC和QOATA

        • QOATA：该模型旨在最大限度提高预算约束下的整体信息质量，作者没有考虑工作者的风险

        • AC：该模型目标是最小化工作者移动的总距离和参与的工作者总数，作者没有考虑信誉和风险

      ​ 评价指标：

      • 任务分配率
      • 每个任务的平均响应时间
      • 每个任务的平均信誉值
      • 每个任务的平均风险值

    ​ 参数设置：两组试验

    ​ （1）当任务数设置为1000时，司机人数从500人增加到2000人，步长为100人；

    ​ （2）当司机数量设置为1000时，任务数量从500逐步增加到2000，步长为100。

    # 第四章 基于边缘计算的任务分配方法

    根据群智系统中挖掘到的影响任务分配的属性，本文建立了一个多目标 优化模型，以求能够分配信誉更高、风险更低以及响应时间更快的工作者来执行 任务。模型包括了一个多目标的优化函数和相关约束条件，并将该模型与已有的 模型进行对比，结果表现良好。但是，在响应时间上，该模型表现不足。此外可 以发现随着数据规模的增加，响应时间也迅速增加。本章将分析这种优化模型的 求解难度，并提出了一种利用移动边缘计算环境的近似求解方法，将其与全局优 化的方法进行对比以验证方法的有效性。

    1. 边缘服务器聚类

      本文将根据边缘服务器的位置对它们进行聚类，使簇心的某个服务器执行分配行为。本文使用的澳大利亚通信数据集，其中包含基站id和基站经纬度等属性。选择其中使用较多的墨尔本CBD区域的基站作为边缘服务器进行聚类实验。

    2. 逐步分配算法

      本节介绍了如何将全局优化的多任务分配问题划分到各个区域并求出近似全局最优的分配结果

      在基于位置的任务分配时，以任务和工作者所在位置为依据划分n个区域，使得 T,...,T 且 W ,...,W ，并对区域内的任务和工作者进行分配，从而在近似最优解的情况下通过减少任务分配算法的执行时间来降低响应时延。

      边缘计算环境中的任务分配方法可以分为三步:首先边缘服务器通过第三章中给 出的多目标优化模型求解INLP，对其覆盖范围内的工作者和任务进行调配;然 后按照上一节中的方法对边缘服务器进行聚类，将相对距离较近的边缘服务器聚 合为一片区域，由位于该区域中心的边缘服务器通过解INLP对聚类区域内的工 作者和任务进行调配，但只考虑仍有余力的工作者和未达到成本上限的任务;最 后由云计算中心通过解同样优化函数的INLP对全局范围内的工作者和任务进行 分配，也只考虑有余力的工作者和未达到成本上限的任务。
